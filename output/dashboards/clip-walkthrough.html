Why this matters: Courtana's API now mixes 500MB full-match recordings with highlight clips. Without the HEAD pre-filter, the analyzer would try to download 500MB files and time out. The _head_size_mb() function sends a lightweight HTTP HEAD request (no download) to check Content-Length before committing.

🔮 Next Frontier: Frame Extraction + Multi-Modal Comparison

The current pipeline sends whole videos to Gemini. The next evolution extracts key frames and runs targeted analysis — enabling paddle brand identification, player tracking, and court mapping at pixel level.

🎥

Frame Extraction

FFmpeg extracts keyframes at serve, speed-up, and celebration moments identified by Gemini timestamps

🔎

Object Detection

YOLO / SAM2 identifies paddle brands, court markings, player positions at pixel precision

🎨

Multi-Modal Fusion

Gemini video + frame-level CLIP embeddings + OCR for text/logos = 3-signal consensus

📊

Spatial Analytics

Court heatmaps, player movement tracking, shot trajectory mapping from frame sequences

🔍

Brand Verification

Cross-reference Gemini's brand calls with actual frame OCR/logo detection for ground truth

🧬

Player Re-ID

Track same player across clips using appearance embeddings — enable longitudinal skill tracking
