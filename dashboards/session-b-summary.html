01 What Was Built

agents/shared.py — Shared Infrastructure

working

Central module all agents import. Clip-journal.db CRUD, corpus scanning (1,280+ files across 15+ directories), UUID extraction (fixed facility-vs-clip UUID bug), Gemini API helpers with cost tracking, ground-truth badge loading.

agents/orchestrator.py — Agent Ensemble Runner

working

Runs all agents in dependency order with fail-soft error handling. Configurable via CLI (--agents, --nightly, --budget). Generates run summaries as JSON + Markdown, plus agent-activity.json for the dashboard. All 4 tested agents completed in 0.3s.

agents/curator.py — Intelligent Clip Selection

working

Replaces FIFO ingest. Fetches from Courtana anon endpoint (4,988 clips found), scores by badge coverage gaps, player diversity, recency, and brand potential. Produces ranked candidate lists with per-clip reasoning. Identified 404 already-analyzed clips and scored all 4,988 remaining.

agents/multi-model.py — Flash vs Pro Comparison

built + dry-run tested

Selects 10% sample of standard-schema clips, runs through both Gemini 2.5 Flash and Pro. Diffs on 9 skill scores, badge predictions, story arc, and dominant shot. Hard budget cap ($5/run). Writes to output/model-agreement/ and journal. Dry-run verified clip selection (762 candidates). Live API test running on 2 clips.

agents/consensus.py — Disagreement Adjudicator

built, awaiting multi-model data

Reads model-agreement files, adjudicates using ground truth (806 clips with badges), confidence scores, and field-type heuristics. Subjective fields (arc, shot type) accepted as valid uncertainty. High-disagreement clips flagged for human review. Outputs verification reports.

agents/validator.py — Self-Consistency Checker

working

Checked 97 clips. Average consistency score: 98.1%. Found 23 clips with issues (all low/info severity). Most common issue: narrative_arc_alignment (22 clips) — the narrative summary doesn't reference the story arc label. Confidence calibration: 5 clips with values, all in high-confidence bucket.

agents/insights.py — Cross-Corpus Pattern Detection

working

Analyzed 1,282 clips across 4 analysis dimensions. Generated 19 raw insights, selected top 7. Discovered real patterns: badge co-occurrence (Consistency King + Dink Machine at 2.0x lift), player archetype clustering (Power Artists vs Hammers), and Court IQ predicting composure under pressure.

agents/prompt-improver.py — Prompt Patch Generator

working

Ingested 10 human feedback entries, 40 badge accuracy patterns, and 0 model disagreements (pending multi-model output). Generated 3 prompt patches: (1) reduce false-positive badges (7 zero-precision badges identified), (2) confidence calibration guidance, (3) general accuracy improvement.

188 co-occurrences vs 93 expected. Lift = 2.0x. Suggests a compound play pattern — players who dink well also tend to be consistent. Could become a "combo badge."

Power-Creativity split reveals two player archetypes

Of clips with power >= 7: only 1 also has creativity >= 7, while 109 have creativity < 4. This defines "Power Artists" vs "Hammers" — a real coaching signal.

04 Architecture Decisions

  • Single shared.py module — all agents import from one place. Prevents drift in UUID extraction, corpus scanning, journal writes. Every agent gets the same view of the corpus.
  • clip-journal.db as single source of truth — SQLite WAL mode for concurrent reads. One row per (clip, agent, timestamp). Idempotency guard prevents double-processing within 24 hours.
  • UUID extraction from filename, not URL — discovered that CDN URLs have facility UUID first and clip UUID second. Filename pattern analysis_{clip_uuid}_{timestamp}.json is the most reliable source.
  • Fail-soft orchestrator — each agent runs in its own try/except. A single clip or agent failure logs and continues. No cascading failures.
  • Budget-aware multi-model — tracks cost per clip and stops when budget is exhausted. Flash + Pro on one clip costs ~$0.02-0.05 depending on video length.
  • Insights agent uses statistical patterns, not LLM calls — pure Python computation over the corpus. Zero API cost, instant results, reproducible.
  • Prompt patches are recommendations, not auto-applied — written as Markdown with human-readable instructions. Bill or the orchestrator decides when to apply.

05 Opportunities Spotted

  • Confidence scale mismatch: Old clips use 1-10 scale, new clips use 0.0-1.0 for analysis_confidence. The insights agent caught this naturally. A migration script should normalize historical values.
  • Curator badge scoring: Some Courtana clips have empty badge_awards arrays but still score high. The scoring could weight badge-rich clips higher once the API returns badge data consistently.
  • Narrative alignment is the #1 consistency issue: 22/97 clips have narrative summaries that don't reference the story arc label. This is a cheap prompt fix — tell the model to mention the arc type in the narrative.
  • Combo badges: The Consistency King + Dink Machine co-occurrence pattern (2.0x lift) suggests a new compound badge. The insights agent could propose these automatically.
  • Player archetype classification: The power-creativity split is clean enough to become a first-class feature — classify every player as Power Artist, Hammer, Finesse, or All-Rounder.
  • Agent scheduling: The orchestrator could become a Claude scheduled task running nightly, producing a morning summary for Bill. Curator runs hourly (before auto-ingest), others nightly.

06 Known Issues / Next Steps

  • Multi-model needs live API validation: Dry-run tested; live run in progress on 2 clips. Once confirmed, consensus agent can adjudicate real disagreements.
  • Curator badge scoring sees empty strings: Some Courtana API badge_awards have empty name fields. The curator scores these as "never tested" (correct behavior but noisy).
  • Self-stability check not yet implemented: The validator has self-consistency and calibration, but not re-analysis stability (would require API calls). Stubbed for future.
  • Narrator agent deferred: Was in the plan as a 7th agent wrapping ElevenLabs. Not built this session — the 6 analytical agents were prioritized.
  • agent-activity.json path resolution: The workbench HTML tries 3 paths for the JSON. May need adjustment depending on gh-pages deploy.
