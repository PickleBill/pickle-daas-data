4What's actively being built next: Track 4 (the agent sprint)

Why this is next: Pre-deciding which "wider accuracy" metrics to track is the wrong call — let agents discover them. Agent outputs also de-risk every other track (Track 2 needs showcase content; Track 3 multi-modal feeds the consensus judge).

The 7 agents:

  • Curator — replaces FIFO ingest with diversity-aware clip selection (brand coverage, arc variety, badge gaps)
  • Multi-model — runs 10% of clips through Gemini Flash AND Pro; logs disagreements
  • Consensus judge — adjudicates Flash-vs-Pro disagreements; flags uncertain clips for human review
  • Validator — self-stability checks (re-analyze, compare ±1pt), self-consistency, confidence calibration
  • Insights — nightly cross-corpus pattern detection; proposes 3-5 NEW metrics per run
  • Prompt-improver — watches human flags → generates targeted prompt patches → A/B tests
  • Narrator — wires the existing ElevenLabs voice pipeline into agent output

Shared infra: output/clip-journal.db — one row per (clip, agent) pair. Single source of truth for what each agent saw and decided.

Budget: $10 Gemini per orchestrator run. Hard cap, idempotent, fail-soft.

Success criterion: Bill opens Product Workbench tomorrow morning and sees a real "Agent Activity" section with curator picks, multi-model disagreements, consensus decisions, validator checks, and proposed new metrics — all generated overnight without prompting.

5Today's pipeline incident — found and fixed

Mid-session diagnostic caught two compounding bugs that had silently stopped new analyses for ~4 hours. Both fixed and verified end-to-end.

Bug 1 — analysis_confidence patch was dead code FIXED

  • The new self-confidence field was added to the in-code ANALYSIS_PROMPT in pickle-daas-gemini-analyzer.py
  • But auto-ingest invokes the analyzer with --prompt-file prompts/v1.X.txt, which loads the prompt from disk — the in-code version is never used in production
  • Fix: Added the field to prompts/v1.3-20260410.txt, bumped DEFAULT_PROMPT in tools/auto-ingest.py from v1.2 → v1.3
  • Verified: 2 fresh test analyses populated analysis_confidence: 0.9 with reasoning

Bug 2 — Courtana feed shifted; every new clip was being skipped FIXED

  • Courtana now serves 500MB full-match recordings interleaved with highlight clips on the anon endpoint
  • Analyzer's hardcoded 20MB inline-upload limit silently skipped every oversized clip
  • Hourly batches were producing manifests with zero analyses — log said "1 new clip" so it looked healthy
  • Fix: Added HEAD pre-filter in fetch_new_clips() with max_mb=18.0; scans deeper into pagination to keep the batch full of viable clips
  • Verified: Test ingest pulled 2 viable clips (49 oversized skipped at fetch); both analyzed cleanly

Lesson logged

Added incident entry to BILL-OS/LESSONS.md so next session knows: (1) prompt patches must go in prompts/v1.X.txt; (2) "no new clips" requires checking dir contents, not log counts.
